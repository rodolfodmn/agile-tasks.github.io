Bis2Crum

Método Scrum adaptado para Bis2bis.

Resumo

Um time composto por devs e um ou mais testers coringas ou não, entregam as features disponibilizadas pelo Gerente de Projeto, onde o time gerencia como as features vão ser desenvolvidas, dividindo as features em tasks, e podendo dividir as features em mais features caso necessário, sendo que uma feature, deve ser um entregavel para o cliente. Cada feature possui as tasks obrigatórias: testes, correções, deploy e ajustes, sendo que os testes são realizados pelos testers e as correções e deploy pelos devs. Os testers devem saber se a feature está entregável ou não, sendo entregavel ela é deployda e disponibiliza para o Gerente de Projeto. Toda semana o time se reúne e valida as features que vão ser desenvolvidas durante a semana, sendo que se hover alguma task da semana passada que não foi entregue ela simplesmente é reanalisada. Caso haja algum bloqueio para algum dev, esse bloqueio é passado para o time, e caso o time não conseguir resolver, é passado para o Gerente de Projeto, sendo função dele resolver o bloqueio.

  • O Gerente de projeto cria o projeto na ferramenta de gerência, e alimenta o projeto com as features referenciadas com o documento de requisitos
  • Reunião de apresentação do projeto, onde o Gerente de projeto da uma visão geral do projeto para o time
  • Reunião do time para analise do entregavel da semana, com a pontuação normal do scrum
  • Desenvolvimento da feature
    • o dev seleciona uma feature para inicar o desenvolvimento
    • o dev seleciona quebra a feature em tasks
    • com todas as taks desenvolvidas, inicia o teste
      • o tester testa e insere os erros encontrados na task correções
      • o dev analisa e aplica as correções necessárias
      • o tester idientificando que a feature está sem erros e funcionando como esperado tesmina a task
    • o dev da feature deploya a feature e passa para o Gerente
    • Caso exista alguma falta na feature o Gerente pode informar as mesmas na task faltas
    • o dev analisa e reinicia o dev da feature até o Gerente aprovar
    • Com a aprovação do Gerente, a taks ajustes é concludia, concluindo o dev da feature
